登录 + 扫码签到：具体操作流程

  1. 登录：用户打开小程序后，通过微信授权或手机号登录；未绑定手机号时需先完成「绑定手机号」。
  2. 绑定：将微信身份与系统内受试者/访客身份绑定（手机号、预约/入组信息），供后续签到与工作台识别。
  3. 扫码签到：到实验室后扫描接待台-大屏投影现场展示的「当日二维码」（大屏/电脑），小程序上报签到事件，接待台队列与叫号系统可识别到实验室状态。
  4. 扫码签出：离开实验室时扫描「当日二维码」，更新到访状态，便于队列与后续访视统计。

上述流程目前处于S8 上线测试：已发布微信小程序正式版，与招募台、接待台进入项目实测。

工作台与登录/绑定的关系与扫码签到/签出的关系
招招·招募台提供预约、入组数据，用于绑定时的身份与项目关联；小程序拉取「我的预约」等。签到后可回写/同步「到实验室」状态，便于招募侧统计到访与依从性。
和序·接待台接待台需识别「谁来了」：依赖登录+绑定后的身份与手机号，用于队列、叫号与现场核验。接待台通过大屏投影提供「当日二维码」；小程序扫码后上报，接待台更新队列、大屏与叫号；核心联动方。

总结：登录+绑定解决「是谁」；扫码签到/签出解决「何时到实验室/何时离开」；招募台侧重预约与入组数据，接待台侧重现场队列与到实验室状态。

与「登录」的关系

登录只解决「你是谁」（微信 + 手机绑定 → Subject）。扫码签到额外需要：有效当日二维码 + 当日签到状态机 +（若要完整业务闭环）当日预约与项目信息一致。 招募台主要在报名→筛选→入组上扩展 Subject/Enrollment；不替代接待侧 SubjectAppointment 的创建，除非其他流程已接好。

闭环验收检查清单（建议顺序）

  1. 接待导入：受试者手机号 = 小程序绑定手机号；当日有 CONFIRMED/PENDING 预约。
  2. 接待台今日队列可见该人。
  3. 小程序同一账号扫当日接待码 → scan-checkin 200 → 队列变已签到；核对 SubjectCheckin、预约是否 COMPLETED。
  4. 再扫一次 → 签出；接待侧签出与看板是否一致（无看板记录时接受「跳过镜像」）。
  5. 若走招募：入组完成后是否仍须再走接待导入/约访——制度断点，代码默认不自动补齐。
  6. 签出后访视预约：接待 H5 已实现；小程序 L8 展示与联调验收可继续对齐。

多项目同日、入组后自动写预约等需单独产品规则；可在需求阶段拆任务改接口。
