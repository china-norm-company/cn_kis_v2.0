# 工作台上线波次规划
> 唯一真相源：分层分波次是公司级上线实施的核心调度依据

**制定时间**：2026-03-26  
**依据材料**：DATA_ACTIVATION_PLAN.md · CN_KIS_AI_VALUE_AND_ROLE_ANALYSIS.md · workstations.yaml  
**负责人**：开发小组负责人  
**更新频率**：每两周复盘一次，随工作台成熟度晋级/降级

---

## 核心原则

不是 18 个工作台"同时上线"，而是分三层推进：

| 层级 | 名称 | 标准 | 推进节奏 |
|------|------|------|---------|
| **Wave A** | 可推广 | 角色价值可见、数据就绪、AI 可用 | 本月全力推广 |
| **Wave B** | 试点陪跑 | 核心数据有、部分场景可用、需陪跑支持 | 本月试点 1-2 个核心角色 |
| **Wave C** | 继续建设 | 数据或功能不足，不宜对外承诺 | 下一阶段建设，不做推广承诺 |

---

## Wave A — 可推广（当前可全力推进）

### 📌 判断依据
- ✅ 画像/结构化数据已发布（受试者 11K + 项目 4K + 人员 434）
- ✅ 飞书邮件/IM/任务已激活进入知识检索（363K 条已发布）
- ✅ 有明确的"第一角色"和"30 秒内可见的价值场景"

| 工作台 | 第一服务角色 | 第一价值场景（30秒内验证） | 数据就绪程度 |
|--------|------------|--------------------------|------------|
| **secretary**（子衿·秘书台） | 总经理 / 管理层 | 早晚报 + 工作台门户 + 飞书信息汇总 | ✅ 全量通信已激活 |
| **research**（采苓·研究台） | 项目经理 / CRC 协调员 | 项目最新进展 + 今日访视安排 | ✅ 4K 项目画像 + 邮件 |
| **recruitment**（招招·招募台） | 招募经理 / 招募专员 | 待跟进受试者清单 + 入组漏斗状态 | ✅ 11K 受试者全生命周期 |
| **admin**（鹿鸣·治理台） | IT 专员 / 系统管理员 | RBAC 配置 + 账号管理 + 审计日志 | ✅ RBAC 已通过验收 |

**Wave A 推广目标**（本月）：
- secretary：总经理/管理层每天使用早报，秘书/助理通过门户管理待办
- research：项目经理和 CRC 协调员以系统为第一信息源（不再查飞书群）
- recruitment：招募专员每日从系统取跟进清单，不用 Excel
- admin：IT 专员通过治理台管理所有账号权限，不用手动配置

## Wave B — 试点陪跑（本月选 1-2 个核心角色试点）

### 📌 判断依据
- ✅ 通信数据已就绪，结构化业务数据有一定积累
- ⚠️ 专业 SOP / 法规类知识尚缺，AI 深度回答有限
- ⚠️ 需要"陪跑"——有 IT 或实施人员辅助，不能完全自服务

| 工作台 | 第一服务角色 | 当前可用场景 | 当前不可承诺的场景 | 核心缺口 |
|--------|------------|-----------|-----------------|---------|
| **execution**（维周·执行台） | CRC 主管 | 团队工单汇总 + 访视历史查询 | 自动合规校验 | 访视执行 SOP 未入库 |
| **quality**（怀瑾·质量台） | 质量经理 | 偏差/CAPA 邮件信号查询 + 历史记录 | GCP 合规判断 + 自动偏差分类 | SOP库和法规库为零 |
| **finance**（管仲·财务台） | 财务经理 | 易快报冲突审核 + 应收账款状态 | 预算预测 + 成本分析 | 财务 SOP 未入库 |
| **digital-workforce**（中书·智能台） | 开发组 / 运营 | 多智能体对话 + 动作回放 | 全自动化业务流转 | 高价值场景配置不足 |

**Wave B 试点策略**（本月）：
- 每个 Wave B 工作台只对 **1 个核心角色的 1 个场景** 试点
- 试点前必须明确告知用户："这个功能目前是辅助工具，不替代判断"
- 每周收集 1 次试点反馈，作为是否晋级 Wave A 的依据

## Wave C — 继续建设（本阶段不做推广承诺）

### 📌 判断依据
- ❌ 专业知识库近乎空白（SOP / 法规 / 操作规程）
- ❌ 结构化数据不完整，AI 无法给出有意义的答案
- ❌ 目标角色对系统的期望是"深度专业辅助"，当前能力不匹配

| 工作台 | 核心缺口 | 建设重点 | 预计晋级条件 |
|--------|---------|---------|------------|
| **crm**（进思·客户台） | 客户画像仅 60 条，商务管线数据稀少 | 补充客户/合同历史数据 | 客户画像 >500 条 |
| **hr**（时雨·人事台） | 人员档案 434 条，无培训/绩效历史 | 接入 HR 系统数据源 | 员工全生命周期档案就绪 |
| **equipment**（器衡·设备台） | 校准/维护 SOP 未入库，设备台账不完整 | 导入设备档案 + 校准 SOP | 设备台账 >80% 覆盖 |
| **material**（度支·物料台） | 库存/批次/效期数据未入库 | 接入物料管理数据 | 批次效期预警可用 |
| **facility**（坤元·设施台） | 场地/环境监控数据缺失 | 接入环境监控系统 | 实时环境数据可读 |
| **evaluator**（衡技·评估台） | 评估 SOP/检测方法库未入库 | 导入检测方法数据库 | 检测方法库 >200 条 |
| **lab-personnel**（共济·人员台） | 排班/工时无历史数据 | 接入排班系统 | 历史工时数据 >3 个月 |
| **ethics**（御史·伦理台） | 法规库（GCP/伦理规范）为零 | 录入 GCP + 国内伦理法规 | 核心法规库 >50 条 |
| **reception**（和序·接待台） | 当日来访实时数据 | 接入预约系统实时数据 | 实时叫号接口就绪 |
| **control-plane**（天工·统管台） | 系统资源监控数据未聚合 | 接入运维监控数据源 | 系统健康面板可用 |

## 工作台晋级/降级规则

### 晋级条件（Wave C → B → A）

```
晋级前必须满足：
□ 第一核心角色已明确
□ 第一价值场景已验证（真实用户使用 > 1 周）
□ 角色在该场景下能 30 秒内得到有用答案（截图存档）
□ 过去 1 周无"数据显示但实际空白"的投诉
□ 对应知识库数据覆盖率 > 60%
```

### 降级触发条件

```
以下情况触发自动降级讨论：
□ 连续 2 周该工作台无真实活跃用户
□ 用户反馈"系统比 Excel 还麻烦"超过 2 次
□ 关键数据源接入中断超过 3 天
□ 发现过度承诺（对用户承诺了 Wave C 工作台的深度 AI 能力）
```

## 每周上线复盘会议结构（每周一 10:00）

```
[10 分钟] Wave A 工作台推广进展
  - 每个台上周活跃用户数 / 提问次数 / 反馈问题
  - 是否有"系统比 Excel 还麻烦"的反馈？原因是什么？

[10 分钟] Wave B 试点反馈
  - 试点角色本周遇到了什么问题？
  - 是数据问题 / 知识问题 / 界面问题 / 流程问题？
  - 是否具备晋级 Wave A 的条件？

[10 分钟] Wave C 建设进展
  - 本周推进了哪个台的数据建设？
  - 下周哪个台最有可能晋级 Wave B？

[10 分钟] 智能体机会池更新
  - 本周发现了哪个场景最适合接入主动洞察/建议型智能体？
  - CodeRabbit / Cursor AI 在开发效率上带来了什么变化？
```
