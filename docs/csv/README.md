## 验证生命周期

```
URS（需求） → FRS（功能规格） → IQ（安装确认） → OQ（运行确认）→ PQ（性能确认）→ 上线
      ↓                                                                              ↑
  CHANGE_CONTROL（变更控制，每次部署新版本都需更新 OQ）─────────────────────────────┘
```

## 责任矩阵（RACI）

| 活动 | 系统所有者 | QA 负责人 | 开发负责人 | IT 负责人 |
|------|-----------|----------|-----------|----------|
| URS 起草 | R | A | C | I |
| FRS 起草 | C | A | R | I |
| IQ 执行 | I | A | C | R |
| OQ 执行 | C | A | R | I |
| PQ 执行 | R | A | C | I |
| 变更审批 | A | R | C | I |

> R=Responsible（执行）A=Accountable（负责）C=Consulted（咨询）I=Informed（知悉）

## GxP 关键要求检查清单

- [ ] 系统访问受控（用户名+密码+飞书 OAuth 双因子）
- [ ] 审计轨迹完整且不可修改（`t_audit_log` 无 DELETE/UPDATE 权限）
- [ ] 电子签名符合要求（`t_audit_log.performed_by` + 时间戳）
- [ ] 数据备份与灾难恢复程序已验证
- [ ] 时钟同步（NTP，误差 < 1 秒）
- [ ] 验证文档由 QA 批准并归档
