# CHANGE_CONTROL — 变更控制程序

**系统**：CN-KIS V2.0  
**文件编号**：CN-KIS-CSV-CC-001  
**版本**：1.0  
**生效日期**：2026-03-21

---

## 1. 目的

定义 CN-KIS V2.0 系统变更的评估、审批、实施和验证程序，确保每次变更不引入新的合规风险或数据完整性问题。

---

## 2. 变更分类

| 类别 | 定义 | 示例 | 验证要求 |
|------|------|------|---------|
| **重大变更（Major）** | 影响核心 GxP 功能或数据结构 | 新增数据模型、更改权限体系、修改认证逻辑 | 重新执行完整 OQ，更新 OQ 版本号 |
| **中等变更（Moderate）** | 影响非核心功能或 UI | 新增工作台页面、修改 API 响应格式 | 执行相关 OQ 测试案例，记录测试结果 |
| **轻微变更（Minor）** | 配置调整、文字修正 | Nginx 参数优化、日志级别调整 | 无需重新验证，记录变更即可 |

---

## 3. 变更控制流程

```
提交变更申请（变更记录表）
    ↓
QA 评估变更影响（分类：Major / Moderate / Minor）
    ↓
    ├── Major：系统所有者 + QA 共同审批 → 安排验证窗口
    ├── Moderate：QA 审批 → 安排测试
    └── Minor：开发负责人审批 → 直接实施
    ↓
实施变更（部署到生产）
    ↓
执行回归测试（见 OQ.md，按分类执行对应测试集）
    ↓
记录测试结果 → QA 批准 → 关闭变更单
```

## 5. 紧急变更程序

当生产系统出现严重安全漏洞或患者安全风险时，可执行紧急变更：

1. 开发负责人立即实施修复
2. 同步通知 QA 和系统所有者
3. 48 小时内补充验证测试
4. 7 个工作日内完成正式变更审批归档
