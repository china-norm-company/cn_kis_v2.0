# Mermaid 流程图作图规范（文档用）

适用于本仓库 `docs/**/*.md` 中的 Mermaid `flowchart` / `sequenceDiagram` 等。

## 0. 强制（易复发问题 · 修改配图前必读）

1. **图中可见文案不得出现英文字符**（含类路径、带 `/` 的路径、`REST`、`GET`、代码字段名、反引号包住的 API 名等）。**技术对照一律放在图外的 Markdown 表格**，不得写进 Mermaid 节点、连线标签、子图标题。
2. **禁止在节点里写 `<span>`、`<div>`、`<br/>`**：部分预览器会把标签**当正文画出来**（含字面量 `br`、引号）。**推荐 `htmlLabels: false`**，节点**单行中文**；换行需求用**更短措辞**或拆节点，勿用 HTML。
3. **文字不得出框（强制）**：`htmlLabels: true` 时才用 **`<br/>` 短行**；**`htmlLabels: false`** 时**单行缩短**，**彩框节点**宜 **≤6 个汉字**；完整语义放**图外表格**。**同一工作台子图内**用 **`rankSpacing` 小** + **`~~~` 透明链**压紧上下块（约 **≤3 个节点高度**）。
4. **步骤与表格一致**：用 **「第1步」「第2步」…全中文**，与正文「图中步骤」列一一对应。

## 1. 同一工作台不要拆成多个并列子图

- **同一飞书工作台 / 同一业务域**（如采苓·研究台、维周·执行台）在一张图里用 **一个外层 `subgraph`** 包住该台所有节点。
- 若需压缩高度，可在 **该 subgraph 内** 再套一层 **`direction LR`** 的内层子图（如并排两个看板节点），**不要**用两个并列的外层子图假装两个工作台。

## 2. 图上只写中文；技术含义进表格

- 节点、连线标签、子图标题：**仅中文**（可用 **①②③④** 圈号表示分区；**步骤用「第N步」**）。**禁止** `/research`、`apps/...`、`REST`、`` `GET` `` 等出现在图中。
- **路由、接口、应用标识** 仅出现在图外 **表格 / 说明摘要**。

## 3. 文字不出框

1. **`htmlLabels: true`** 时，节点用 **双引号**；**优先不用** `<div style=...>`（避免渲染差异与源码噪音）；用 **`<br/>` 主动断行**（每行约 6～10 个汉字为宜）。
2. **字号**：一般文档 **`themeVariables.fontSize`：16px～18px**。《小程序日记开发整体需求_图》图 1：当前示例为 **框内 11px**、侧栏约 **30px**（`htmlLabels: false`，见下）；若用 `htmlLabels: true` 可参考 **框内 18px**、侧栏 `<span>` **54px**，见 §4 下说明。
3. **`flowchart.nodeSpacing` / `rankSpacing` / `padding`**：随字号略增，避免贴边；**不要盲目加大 rankSpacing** 拉高整图。

## 4. 初始化参数（flowchart 参考）

```text
%%{init: {'flowchart': {'htmlLabels': true, 'nodeSpacing': 26, 'rankSpacing': 32, 'padding': 14, 'useMaxWidth': true}, 'themeVariables': {'fontSize': '17px'}}}%%
```

**「第N步」与紧凑布局**（`小程序日记开发整体需求_图.md` 图 1）：**`htmlLabels: false`**；**`第N步 --- 彩框`** 横向直连（`nodeSpacing` 取小，如 8～10）；步骤字号用 **`style … font-size`** 约为框内 **3 倍** + `color` 浅色；模块内 **`~~~` 竖链** + **`linkStyle` 透明**压层距；**业务边**单独着色。步骤编号须与其中「步骤说明」表**工作流顺序**一致。

**三列版面**（同上）：根 **`flowchart LR`**，**左 `COL_LEFT` · 中 `COL_MID` · 右 `COL_RIGHT`**。左列内 **`direction TB`**：先 **研究台子图**，其下 **小程序子图**（同一左列上下叠放，宽度由引擎尽量拉齐）；**后端居中**、**执行台居右**。列序可用各列顶部的 **透明锚点**（如 `la` / `ma` / `ra`）及 **`la ~~~ ma`、`ma ~~~ ra`** 加固。**勿画 `F --> B` 一类「执行台指回后端」的有向边**：在 dagre LR 下易把 **F 排在 B 左侧**，执行台会跑到后端左边；督导写后端的语义改由**表内说明**，或仅用**无向 `~~~`**（慎用，勿与有向边构成难读图）。若需 **中列与右列竖向体量接近**，可在中列 **B 与 E 之间**加 **透明占位节点**并用 **`~~~` 串联；改边后须重算 **`linkStyle` 下标**（顺序一般为：左列全部内边 → 中列内边 → 右列内边 → 列间脊线 → 跨列业务边）。顶底齐平为**示意**，以正文结构说明为准。

## 5. 与产品文档的关系

业务细节以正文表格为准；图只表达**角色、顺序、数据去向**。**自动换行**不可靠，以 **`<br/>`** 为主。
