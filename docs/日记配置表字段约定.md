# 日记配置表 — 字段约定与中英对照

> **用途**：存放**日记配置类**落库形态定稿，供产品、后端、对接同事统一口径。  
> **关联**：[`小程序日记功能2.0.md`](./小程序日记功能2.0.md)（总需求与试点）、[`DATABASE_STANDARDS.md`](./DATABASE_STANDARDS.md)（表命名规范）。  
> **实现**：以 Django 迁移与 OpenAPI 为准；**本表在仓库内需新建**（见下「仓库现状」），通过 **Model + `makemigrations` + `migrate`** 落库后再灌数据。

---

## 〇、仓库现状：配置表已落地（以迁移为准）

以下为 **CN_KIS_V1.0** 后端实现（若后续调整模型，以 `apps/subject/migrations/` 为准）：

| 对象 | 说明 | 是否可当作「日记配置表」延用 |
|------|------|------------------------------|
| **`t_subject_diary`（`SubjectDiary`）** | 存**日记条目**（受试者每次填写内容） | **否**。这是**条目表**，不是配置表。 |
| **`t_questionnaire_template` 等** | 通用**问卷模板**，含 `form_definition` 等 | **否**。职责为问卷体系，无本文件约定的 **`project_id` + `rule_json` + 日记门禁** 等一体化结构；若产品决定「日记完全走问卷模板」，属**另案设计**，与本表**不是**简单延用同一张表。 |
| **`t_subject_diary_config`（`SubjectDiaryConfig`）** | 项目级日记配置 | **是**。模型见 `backend/apps/subject/models_diary_config.py`；受试者拉取 **`GET /api/v1/my/diary/config?project_id=`**（需已发布且 `researcher_confirmed_at` 非空）。试点灌库：**`python manage.py seed_diary_config_pilot`**。 |

**结论**：配置类以 **`t_subject_diary_config`** 为准，字段以 **§三** 为准。条目侧 **`diary_config_id` / `answers` JSON** 等见 **§四**，随 B3 迭代扩展 `SubjectDiary`。

## 一、命名约定（避免 scheme / schema 混用）

| 英文用词 | 中文含义 | 说明 |
|----------|----------|------|
| **`*_config*`** | **配置** | 表名、外键名使用 **config**，表示**一整条配置记录**（按项目、按版本）。 |
| **`form_definition_json`** | **题目与表单定义** | 仅描述**题怎么出、表单结构**；**不用** `schema_json` 作列名，以免与「配置」混淆。 |
| **`rule_json`** | **填写与校验规则** | 应填日期范围、每日提交时间窗、补填等；服务端据此校验（如**窗外拒绝**）。 |
| **`project_id` / `project_no`** | **项目 ID / 项目编号** | **`project_id`** = 项目表主键（外键）；**`project_no`** = 业务编号（如 W26000000）。 |

---

## 二、推荐表名（新建）

| 推荐表名（英文） | 中文含义 | 说明 |
|------------------|----------|------|
| **`t_subject_diary_config`** | **受试者日记配置表** | 一行 = 某一**项目**下、**某一版**的「题目定义 + 规则 + 发布与确认」。**需通过迁移新建**；**不是**已有 `t_subject_diary` 改名而来。 |

Django 模型类名建议与表名对应（如 `SubjectDiaryConfig`），具体以团队规范为准。

---

## 三、字段清单（中英对照）

| 英文字段名 | 中文含义 | 类型与备注 |
|------------|----------|------------|
| **`id`** | **配置记录主键** | 整数，自增；本表主键。**不是**项目编号。 |
| **`project_id`** | **项目 ID（外键）** | 整数 → 项目表主键（如 `project_full_link_project.id`，以实际模型为准）。 |
| **`project_no`** | **项目编号** | 字符串，可选冗余；如 `W26000000`，便于导出与人工核对。 |
| **`config_version_label`** | **配置版本标签** | 字符串，可选；如 `v1`、`2026Q1`；非必须。 |
| **`form_definition_json`** | **题目与表单定义（JSON）** | 各题 **`id`**、题型、选项、顺序、是否必填等；小程序**仅依赖本列**渲染表单。 |
| **`rule_json`** | **填写与校验规则（JSON）** | 应填起止日、每日提交时间窗、时区、补填天数等。 |
| **`status`** | **配置状态** | 枚举，如 `draft`（草稿）/ `published`（已发布）；与门禁配合。 |
| **`researcher_confirmed_at`** | **研究员确认时间** | 日期时间，可空；**非空**表示 **2.0 配置门禁**已满足（本版可对受试者生效）。后端模拟灌库时可写入当前时间。 |
| **`supervisor_confirmed_at`** | **督导确认时间** | 日期时间，可空；**3.0** 若需督导二次确认再使用。 |
| **`create_time`** | **创建时间** | `auto_now_add`，与项目规范一致。 |
| **`update_time`** | **更新时间** | `auto_now`，与项目规范一致。 |

## 四、与日记条目表的关联

日记**条目**表（如扩展后的 `t_subject_diary`）建议包含：

| 英文字段名 | 中文含义 |
|------------|----------|
| **`diary_config_id`** | **日记配置 ID（外键）** → 本表 **`id`** |
| **`project_id`** | **项目 ID**，与配置表一致 |
| **`answers`** | **答案（JSON）**；键与 **`form_definition_json`** 中题目 **`id`** 一致 |

**唯一约束（建议）**：`(subject_id, project_id, entry_date)`；一日多份时增加槽位字段后再调整，见 [`小程序日记功能2.0.md`](./小程序日记功能2.0.md) §2.3.4。

```json
  "timezone": "Asia/Shanghai",
  "diary_period": { "start": "2026-03-23", "end": "2026-04-23" },
  "fill_time_window": { "start": "09:00", "end": "18:00" },
  "frequency": "daily",
  "retrospective_days_max": 7,
  "late_reason_required_when_retrospective": true
}
```
